# Cross Site Scripting (XSS) Attack - DVWA

## Introduction

Cross-Site Scripting (XSS) is a vulnerability that allows attackers to inject malicious scripts into web pages viewed by other users.

## Objective

The objective of this task was to demonstrate how an attacker can inject JavaScript code into a vulnerable web application.

## Tools Used

* Kali Linux
* DVWA
* Web Browser

## Steps Performed

1. Logged into DVWA.
2. Set the security level to **Low**.
3. Navigated to the **XSS (Stored)** section.
4. Entered the following script in the message field:

<script>alert('XSS')</script>

5. Clicked **Sign Guestbook**.

## Result

A popup alert message appeared in the browser, showing that the JavaScript code was executed successfully.

## Conclusion

XSS vulnerabilities allow attackers to run malicious scripts in a victim’s browser, which can lead to session hijacking, data theft, or website defacement.
