SQL Injection is one of the most common web application vulnerabilities where an attacker can manipulate the database by inserting malicious SQL queries into input fields. This usually happens when a web application does not properly validate or sanitize user input before using it in database queries. As a result, the attacker can bypass authentication or access sensitive data stored in the database.

In this project, SQL Injection was performed using the DVWA platform. The attack was carried out by entering a specially crafted input such as ' OR '1'='1 in the input field. This input changes the logic of the SQL query in such a way that the condition always becomes true. Because of this, the application returns all user records or allows login without proper credentials. This demonstrates how weak input validation can lead to serious security issues.

The impact of SQL Injection is very serious because it can allow attackers to access confidential data, modify database records, or even delete important information. To prevent this vulnerability, developers should use prepared statements, parameterized queries, and proper input validation techniques. These methods ensure that user input is treated as data and not as part of the SQL command.
